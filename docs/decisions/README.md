# Architecture Decision Records

This directory contains ADRs for NovaPay — permanent records of
significant architectural decisions: what was decided, what was
rejected, and why.

## Format: MADR (Markdown Any Decision Records)
## Status values: Proposed | Accepted | Deprecated | Superseded by ADR-NNN

Each ADR answers three questions:
1. What situation forced a decision?
2. What was decided, and what was explicitly rejected?
3. What does the system permanently gain and give up?

ADRs are referenced in CLAUDE.md as architectural constraints.
Claude Code must not propose changes that violate an accepted ADR
without explicitly flagging the conflict and getting confirmation.
